result of running the Linear Regression algorithm

The Linear Regression algorithm is a machine learning algorithm that works using a linear line

The algorithm tries to find a line that best fits all of the points

a line is defined by the equation y=mx+c

The "m" is called the coefficient and the "c" is called the intercept

The coefficient is the slope of the line, and the intercept determines how high or low it is (movement of the line on the y axis)

The model picks a line and tries to find the quickest way to go to the best line

How does the algorithm do that you ask?

The algorithm calculates what we call the "loss" function.

The loss function measures how bad the model predictions are

The loss function calculates the distance of the correct points to the predictions

We calculate the euclidean distance by the formula (x1-x2)^2+(y1-y2)^2

The loss function is the sum of all of the distances

The Linear Regression algorithm tries to minimize the loss function.

We determine how to decrease the loss function by something called "Gradient Descent"

Gradient Descent is a concept in calculus used to calculate the gradient, the upwards direction, and then takes the opposite direction, to get the downwards direction

You can think of following the gradient descent as a ball rolling down a hill. Like shown below:

We calculated the Gradient Descent for both the coefficient and the intercept, this way we are decreasing them both

An important thing to note is that we need to define a learning rate, which is a constant our function multiples the Gradient Descent to take a step in that direction

We are trying to find a local minima, a point where the function converges to some minimum.

It's really, really difficult to find the best minima but is really easy to find a local minima which most of the time provide us with a good enough solution

Why is the learning rate important to find the local minima? Because if our learning rate is too extreme we can have all sort of problems!

If our learning rate is too big, we will encounter a problem called Over-shooting.

Over-shooting is when our learning rate is too big, so when we try to reach our local minima we go too far, and when go back again and on and on

If our learning rate is too small, our algorithm will take a lot of time to run and require a lot of non-essential runtime. this is called under-shooting

When we have our best found linear line at (or close) to a local minima we can use it's equation to predict the future or the past.
